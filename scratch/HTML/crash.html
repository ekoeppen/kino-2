1. Einleitung

KinoView dient zum Bearbeiten und Verwalten von XML Dokumenten. Im folgenden wird die Funktionsweise des Programm näher erläutert. Für tiefergehende Informationen verweise ich auf die Diplomarbeit.

2. Voraussetzungen

KinoView wurde mit Hilfe von Wafe programmiert. Es muß mindestens die Version 1.0.19 installiert sein. Das Programmpaket ist unter http://www.wi-inf.uni-essen.de zu bekommen.

Desweiteren verwendet KinoView das Kino-Widget. In der oben beschriebenen Distribution von Wafe ist das Kino zwar enthalten, allerdings noch nicht vollständig. Es muß mindestens die Version 2.11.10 enthalten sein. Ist dies nicht der Fall kann die Kino-lib auch nachträglich eingebunden werden. Im Programm ist ein entsprechender Aufruf enthalten.
Die Kino-Lib: m-kino2Gen.so muß in dem im Quellcode stehenden Verzeichnis sein. Momentan lautet dies:

/usr/X11R6/lib/X11/wafe/tcllib/bin/


Die entsprechende Zeile ist ganz oben im Quellcode enthalten und kann leicht mit einem Editor angepaßt werden. Sollte Kino in einer späteren Wafe-Version komplett enthalten sein, kann diese Zeile auskommentiert werden.

3. Bedienung

3.1 Menü - Datei

3.1.1 Neuer Baum

Der aktuelle Baum und der Inhalt des Kino-Widgets werden gelöscht.

3.1.2 Datei laden

Ermöglicht das laden von lokal gespeicherten Dateien. Korrespondierend kann der Button "<<---- Laden" unterhalb des Dateiauswahlfensters gedrückt werden. Unter "Selection" ist die gewünschte Datei einzutragen.

3.1.3 Speichern

Speichert eine Datei unter dem Namen im Feld "Selection". Korrespondierend kann der Button unterhalb der HTML-Ausgabe gedrückt werden.

3.1.4 Nur als Text speichern

Speichert nur den Text des aktuellen Dokumentes. In dem folgendem Dialog kann der gewünschte Dateinamen eingetragen werden.

3.1.5 Netzdatei laden

Ist die Datei nicht lokal sondern im Netz abgelegt, so kann man diese Funktion benutzen. Im Dialog muß die Adresse des Quellrechners und der Name der Datei eingetragen werden.

3.1.6 Netzdatei speichern

Ebenso wie das Laden über ein Netzwerk ist es auch möglich eine Datei über das Netzwerk zu speichern.

3.1.7 Beenden

Beendet die KinoView. Korrespondierend kann der Button "Beenden" unterhalb der HTML-Ausgabe gedrückt werden.

3.2 Bearbeiten

3.2.1 Baum erneuern

Der Baum wird gelöscht und neu aufgebaut.

3.2.2 Baum löschen

Der Baum wird gelöscht. Der Inhalt des Kino-Widgets bleibt allerdings bestehen.

3.2.3 PData kopieren

Kopiert das aktuell ausgewählte PData-Element inklusive eventueller Kinder.

3.2.4 PData ausschneiden

Schneidet das aktuell ausgewählte PData-Element inklusive eventueller Kinder aus.

3.2.5 PData einfügen

Man kann ein PData-Element auf zwei Arten einfügen:

  1. Einfügen als Kind einer PDataBox
  2. Einfügen innerhalb des selben Astes

Fügt man das Element als Kind einer PDataBox ein, muß man vorher die gewünschte Box selektieren. Nach anwählen der Funktionen wird das Element als letztes Kind eingefügt.

Wird das Element innerhalb des selben Astes eingefügt, so wird es oberhalb des ausgewählten PData-Elements eingefügt.

3.2.6 PData sichern

Es ist auch möglich den Zwischenspeicher lokal zu sichern. Das PData-Element wird unter den im "Selection" Feld bezeichneten Namen gespeichert.
Korrespondierend kann auch der Button Zwischenspeicher sichern gedrückt werden.

3.2.7 PData löschen

Das aktuell ausgewählte PData (inklusive eventueller Kinder) wird gelöscht.

3.2.8 Neues PData

Ein neues PData-Element wird oberhalb des gerade ausgewählten Elements eingefügt. Es können folgende PData-Elemente eingefügt werden:

  1. PDataBox
  2. PDataText
  3. Linebreak

3.2.9 Neues Kind

Ein neues PData-Element wird als letztes Kind der ausgewählten PDataBox eingefügt. Es können folgende PData-Elemente eingefügt werden:

  1. PDataBox
  2. PDataText
  3. Linebreak
Achtung: Es muß eine Box ausgewählt sein.

Hinweis: Zum derzeitigen Zeitpunkt ist es noch nicht möglich Insets einzufügen, da Kino dieses noch nicht unterstützt.

3.3 Optionen

3.3.1 Refresh

automatisch: Der Baum wird nach jeder Manipulation automatisch aufgebaut. Dies ist die Defaulteinstellung.

durch User: Der Benutzer muß den Neuaufbau des Baumes durch Baum erneuern selber auslösen.

Hinweis: Bei einer sehr hohen Anzahl an Elementen im Baum kann eine Löschung des Baumes sehr langsam sein. Dies ist ein Problem von X Windows und hat nichts mit dem KinoView zu tun. Sollte es zu erheblichen Verzögerungen kommen, sollte man den Baum nach jeder Aktion selbst erneuern. Dies kann im Einzelfall zu einer erheblichen Beschleunigung führen.

3.3.2 Details

Detail: Es wird ein Teil des Inhalts von PDataText-Elementen angezeigt. Dies soll die Navigation durch den Baum vereinfachen.

Outline: Die Inhalte werden nicht angezeigt. Dies ist die Defaulteinstellung.

3.3.3. Fenster

In diesem Menü kann die angezeigten Fenster ein- und ausgeschaltet werden.

3.4 Weitere Bedienungselemente

3.4.1 Preview

Zeigt ein Preview der unter "Selection" stehenden Datei.

3.4.2 Kopieren

Kopiert die unter "Selection" stehende Datei in den Zwischenspeicher. Der Inhalt der Datei kann danach in einen bestehenden oder neuen PData-Baum eingefügt werden.
